# Jimmy Choo 品牌前端设计规范

本文档是该数据分析系统的**设计蓝图与决策记录**，基于对 [Jimmy Choo 官方网站](https://www.jimmychoo.com/hk/en_HK/home) 和品牌内部 OTB Full Deck 文档的深度解读而来，指导 `CategoryAnalysis.tsx` 及全局样式的 UI 重构。

---

## 1. 色彩规范 (Color Palette)

### 核心概念：一抹金，极度克制

奢侈品设计的色彩哲学不是"去色"，而是"用色有据"。通过研究 Jimmy Choo × AIZAN OTB Full Deck 品牌内部汇报文档，我们提炼出其完整的色彩体系：

> PPT 文档中，页面背景大面积留白，主体文字为近黑色，**唯一的彩色点缀是一种深哑香槟金**，仅出现在章节序号（"01 /"、"02 /"）、关键 Tag 背景（如"SKU OPTIMIZATION MODEL"）以及标题中的强调词（如"MILESTONES"）。这种极度克制的"一抹金"，才是高端奢侈品设计的核心色彩语言。

#### Design Tokens（已落地进 `tailwind.config.ts`）

| Token | 色值 | 用途 |
|---|---|---|
| 近黑主色 | `#1A1A1A` | 主标题、主要正文。非纯黑，略带温度，视感更柔和高级 |
| 纯白 | `#FFFFFF` | 页面与卡片背景，大面积留白 |
| **品牌香槟金** | **`#C5973F`** | **唯一彩色，极度克制。详见下方使用规则** |
| 金色浅底 | `#EFE0C0` | 金色的极浅衬底，用于 hover 背景等辅助场景 |
| 暖米灰分割线 | `#E8E4DF` | 表格行间分割线，比冷灰更贴合皮革/纸张质感 |
| 辅助灰 | `#5A5A5A` | 次要说明文字、表格辅助标注 |
| 极浅暖白 | `#F9F8F6` | 表格行 Hover 态底色 |

#### 品牌香槟金 `#C5973F` 使用规则

**取色来源**：直接从 Jimmy Choo 品牌内部 PPT Deck 中取色，是团队文档使用的官方强调色。

**✅ 允许使用的场景**（少即是多）：
- 维度区块的序号标注（如"04 /"），致敬 PPT 的"01 / 02 / 03 /"格式
- Share Bar 进度条填充色（页面中唯一的数据可视化彩色元素）
- Tab 栏选中指示线（底部细线，2px）
- 弹窗中年份切换按钮选中态
- Header 年份标签徽章
- 商品卡片"OWN"品牌标签（金底白字）
- 本品"JC"标识格（金底白字）

**❌ 禁止使用的场景**：
- 大面积背景块
- 正文或说明文字
- 分割线
- 非核心强调的任何其他场景

## 2. 排版与字体 (Typography)

### 核心概念：全大写 + 宽字间距 = 高级感

字体的选择和排版是奢侈品牌官网的灵魂，注重几何感和呼吸感。

*   **Jimmy Choo 官网专属字体（版权字体，仅供参考）**：
    *   **Franklin Gothic（无衬线体）**：含 `FranklinGothicDemi`（粗体）和 `FranklinGothicBook`（常规）。用于导航、标题、按钮和核心数据，硬朗且具有现代建筑感。
    *   **Life（衬线体）**：含 `LifeRoman`、`LifeItalic`、`LifeBold`。用于正文内容，增添历史手工感与柔美度。
    *   格式：均以 `.woff2` 格式加载，托管于品牌专属 CDN：`https://www.jimmychoo.com/on/demandware.static/Sites-jchgb-Site/-/default/...`

*   **本网站开源平替方案（已落地）**：
    *   **无衬线（替代 Franklin Gothic）**：`Inter` via Google Fonts。几何美感强，全大写后建筑感极佳。
    *   **衬线（替代 Life）**：`Playfair Display` via Google Fonts。笔画粗细对比强，Editorial 质感。
    *   系统保底：`"Helvetica Neue", Helvetica, Arial, sans-serif`

*   **排版规则**：
    *   标题、导航、Tab、表头：**全大写（`uppercase`）+ 宽字间距（`tracking-wider` / `tracking-widest`）**
    *   字重：倾向 `font-light`（300）或 `font-medium`（500），极少使用 `font-bold`
    *   数据字号：略小（`text-xs` / `text-sm`），但必须保证清晰，数字列使用 `tabular-nums` 确保对齐

*   **自定义 Tailwind 字间距扩展**：
    *   `tracking-luxury`：`0.15em`，用于维度标题、商品品牌名
    *   `tracking-luxury-wider`：`0.25em`，用于页面主标题和章节序号

## 3. 组件样式 (Components)

### 核心概念：Editorial Design，摒弃 Web 2.0 圆润感

*   **直角原则（Sharp Corners）**：全站禁止 `rounded-lg` 及以上的圆角。所有按钮、标签、卡片、弹窗一律直角（`rounded-none`）。圆角感等于失去"奢侈品专业感"。

*   **Tab 切换**：下划线式样（金色 2px 底部线），非背景色块，更轻盈。全大写，`tracking-luxury`。

*   **商品卡片（ProductCard）**：
    *   无边框、无阴影，卡片区分纯靠**留白**。
    *   图片比例：**3:4 纵向（`aspect-luxury`）**，致敬奢侈品官网标准比例。
    *   Hover：图片微缩放（`scale-105`，500ms 过渡），增添交互细节感。

*   **数据表格（DimensionTable）**：
    *   **仅保留水平分割线**，去除全部垂直分割线。
    *   表头：`uppercase tracking-luxury text-[10px] text-gray-400 font-semibold`
    *   行高增大（`py-5`），让数据有呼吸感，Hover 高亮使用极浅暖白（`#F9F8F6`）。
    *   维度区块标题：左侧金色序号"04 /"，展开/收起的"+"号动画。

*   **弹窗（ProductModal）**：
    *   遮罩：`rgba(0,0,0,0.6)` 深色遮罩。
    *   弹窗体：纯白直角，无圆角，最大宽度 `max-w-6xl`。
    *   商品格：5 列（`lg:grid-cols-5`），横向间距 `gap-x-6`，纵向行间距 `gap-y-10`，布局疏朗。

## 4. 空间与布局 (Spacing & Layout)

*   **留白（Whitespace）**：提升高级感的核心手段。各维度模块之间间距 `mb-12`，表格行内 `py-5`。
*   **宽屏优化**：容器最大宽度 `max-w-7xl`（代替原来的 `max-w-5xl`），给数据和图片更充裕的呼吸空间。
*   **双栏布局**：分析表格（左侧主区 `flex-1`）+ 图例说明区（右侧 `lg:w-64` 固定栏），保持主分析区的纯净。
*   **对齐**：严格左对齐或中轴对称，避免视觉焦点跳跃。

---

## 5. 品牌 Logo 资产约束 (Brand Assets Constraints)

*   **分辨率瓶颈**：当前收集到的 36 个竞品及买手店品牌 Logo，多数分辨率在 `128x128` 至 `256x256` 之间的 PNG，为避免放大导致的锯齿和模糊感，**渲染尺寸严格控制在 32px 以下**（商品卡片中使用 `w-3 h-3`，约 12px）。
*   **降级方案**：无 Logo 的品牌，自动显示品牌名称首字母组合（Initials）的灰色方块，与 Logo 等大，视觉风格一致。
*   **灰度处理**：所有 Logo 图片统一加 `grayscale` CSS 滤镜，配合单色主题，防止彩色 Logo 与克制的页面基调冲突。
*   **文件存放**：所有 Logo 本地化于 `/public/brands/`，避免外部链接失效风险。
